4.
ошибка "Missing name for resource" - отсутствует имя ресурса
В terraform у каждого ресурса должно быть ДВА имени:
- тип ресурса (например docker_image)
- собственное имя ресурса (например my_nginx_image)


ошибка "Invalid resource name" - неправильное имя ресурса
- имена ресурсов в terraform не могут начинаться с цифры
- в коде имя ресурса "1nginx" начинается с цифры 1, имя должно начинаться с буквы или подчёркивания (_)

ошибка "Reference to undeclared resource" - обращение к несуществующему ресурсу
- в этой строке мы пытаемся использовать ресурс с именем docker_image.nginx, но в нашем коде мы создали ресурс с именем docker_image.nginx_image (с "_image" в конце)
terraform не находит ресурс с именем "nginx" - отсюда ошибка

ошибка "Reference to undeclared resource" - Опечатка в имени ресурса
- мы пытаемся использовать ресурс random_string_FAKE, но в коде у нас объявлен random_string, даже если бы имя было правильное, у нас опечатка: resulT вместо result (в конце должно быть "t", а не "T")

Опасность -auto-approve:
Пропускает подтверждение действий (может случайно удалить/изменить ресурсы).
Полезен в CI/CD (например, в GitHub Actions), где нет интерактивного ввода.

1. Создайте в облаке ВМ. Сделайте это через web-консоль, чтобы не слить по незнанию токен от облака в github(это тема следующей лекции).     environment:
```
      - "MYSQL_ROOT_PASSWORD=${...}"

```
